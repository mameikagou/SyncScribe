1. 检索精度的进化：混合检索 (Hybrid Search) 现状： 只用向量搜索（语义相似度）。

缺点： 用户搜专用名词（比如“Prisma
7”），向量可能觉得它跟“数据库工具”很像，但搜不到精确匹配。

理想状态： 向量检索 + 关键词检索 (BM25)。

先用关键词捞一波，再用向量捞一波，加权合并。这是目前工业界的标配。

2. 结果的优化：重排序 (Rerank) 现状： 数据库返回 Top 5，直接喂给 LLM。

缺点： 数据库认为相似的，可能并不是真正回答问题的。

理想状态： 引入 Rerank 模型 (如 Cohere Rerank)。

数据库先捞 Top 50 -> Rerank 模型精读这 50 条，挑出最相关的 Top 5 -> 喂给 LLM。

这能让回答准确率提升 20% 以上。

3. 知识的结构化：引用与溯源 (Citations) 现状： AI
   回答了，但用户不知道它参考了哪段。

理想状态： 精准溯源。

AI 回答：“Prisma 7 移除了 Rust 依赖 [1]。”

鼠标悬停 [1]，高亮显示左侧文档里的具体段落。

实现： 需要在数据库里存切片的 page_number 和 text_index。

4. 交互的进化：Generative UI 现状： 只是文字聊天。

理想状态： 按需生成界面。

搜财报 -> 生成 K 线图。

搜代码 -> 生成可运行的 Code Sandbox。

这就是 Vercel AI SDK streamUI 的用武之地。
