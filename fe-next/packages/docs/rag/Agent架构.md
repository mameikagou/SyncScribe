### 一、 核心知识点提炼

#### 1. 什么是 Agent？（定义）

- **关键词：** 感知 (Perceive) -> 认知 (Cognition) -> 行动 (Act)。
- **核心：**
  它不再是被动的问答机，而是能够**主动理解环境、制定目标并执行**的系统。

#### 2. Agent 的五大核心器官 (Architecture Flow)

图片里的那个架构图非常经典（类似于 Lilian Weng 的博文配图），它把 Agent 拆解为：

- **感知 (Perception):**
  眼睛和耳朵。负责把环境信息（用户输入、API数据、传感器信号）变成 Agent
  能懂的信号。
- **大脑 - 认知与记忆 (Cognition & Memory):**
  - **Memory:** 短期（对话上下文）+ 长期（向量数据库）。
  - **Planning:** 把大目标拆解为小步骤。
  - **Decision Making:** 决定下一步该干嘛。
- **手脚 - 行动 (Action):** 调用工具（Tools）、输出结果。
- **学习 (Learning):** 通过反馈（Feedback Loop）更新记忆或优化策略。

#### 3. 四种主流架构模式

这是面试中区分初级和高级开发者的分水岭：

- **a) 反应式 (Reactive):** 膝跳反射。看到 A 就做 B。没有记忆，不思考历史。
  - _例子：_ 简单的自动回复机器人。
- **b) 推理式 (Deliberative):** 深思熟虑。有内部世界模型，先规划再行动。
  - _例子：_ 你的 `SyncScribe`（先查资料，再分析，最后写报告）。
- **c) 混合式 (Hybrid):** 反应式 + 推理式。
  - _例子：_ 自动驾驶（遇到障碍物立刻刹车=反应式；规划导航路线=推理式）。
- **d) 学习型 (Learning):** 越用越强。引入强化学习 (RL)。

### 二、 面试/实战应用：如何把这些理论映射到你的项目？

当面试官问你：“请介绍一下你的 Agent 架构”，如果你只说“我用了 OpenAI
API”，那就太单薄了。

你可以直接套用图片里的理论框架，这样说：

#### 话术 1：关于整体架构 (Mapping the 5 Modules)

> **面试官**：你的 `SyncScribe` 架构是怎样的？
>
> **你**： “我的项目采用了经典的 **推理式 Agent 架构 (Deliberative
> Architecture)**，包含完整的感知-认知-行动闭环：
>
> 1. **感知层 (Perception)**：我通过 Next.js 前端收集用户的多模态输入（文本 +
>    文件上传）。
> 2. **认知层 (Cognition)**：这是核心。我利用 **Vercel AI SDK**
>    作为推理引擎，结合 **System Prompt** 进行任务规划（Chain of Thought）。
> 3. **记忆层 (Memory)**：我设计了 **混合记忆系统**。短期记忆使用 Vercel 的
>    Context Window，长期知识库通过 **Prisma + pgvector** 实现了 RAG 检索。
> 4. **行动层 (Action)**：我封装了一套 **Tools
>    (工具箱)**，包括联网搜索、股票查询等，让 Agent 具备与外部世界交互的能力。”

#### 话术 2：关于设计模式 (The Reasoning Loop)

> **面试官**：你的 Agent 是怎么思考的？
>
> **你**： “我采用了 **ReAct (Reason + Act)** 模式。
> 当用户提出一个复杂问题（如‘分析英伟达财报’）时，我的 Agent
> 不会直接瞎编，而是进入一个循环：
>
> 1. **Reason (思考)**：‘我需要先获取财报数据’。
> 2. **Act (行动)**：调用 `query_knowledge_base` 工具检索数据库。
> 3. **Observe (观察)**：拿到检索到的文本片段。
> 4. **Response (响应)**：基于事实生成最终的分析报告。
>    这种架构保证了回答的**事实性 (Groundedness)**，解决了大模型的幻觉问题。”

#### 话术 3：关于未来规划 (BDI 模型)

> **面试官**：未来有什么优化方向？
>
> **你**： “目前主要是被动的推理。未来我想引入 **BDI (Belief-Desire-Intention)**
> 模型的设计理念。 让 Agent 拥有**主动性
> (Proactivity)**。比如它监测到（Belief）用户持仓的股票大跌，虽然用户没问，但它基于‘保护用户资产’的目标（Desire），会主动推送（Intention）一条风险预警。这将把产品从‘工具’升级为‘管家’。”
