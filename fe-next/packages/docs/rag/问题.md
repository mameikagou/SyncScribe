**RAG**

- 我面试过不少人，说熟悉 RAG，结果一问就穿帮。

  LangChain，上个向量库，把chunk和embedding丢进去跑个检索。

- 但只要你简历上写了，面试官就会问你下面这些(当然不写也不一定逃得过)：
  1. Chunk是怎么切的？固定？语义？还是自适？

  3. Rerank用没用？怎么融合 BM25 和dense 检索？

### 第七张图片：RAG 基础解释与推荐系统类比

- RAG的前半段几乎就是推荐系统那套召回+排序+精排的逻辑：embedding=向量化特征建模检索=多路召回
  rerank=打分排序
- 但后半段还多了 prompt 设计、上下文拼接、生成模型行为控制几个大坑。
- **基础解释：**
  1. Chunk按标点和长度切,最普通。
  2. 普通文档场景,bge-v3够用吧。
  3. bm25和dense的结果一把扔给rerank?
  4. Prompt撰写有什么特殊坑点吗?暂时没想到。
- 上下文拼接：召回的是chunk，拼接的是chunk对应的content。如何界定chunk对应的content？依赖一个好的文档结构解析。
- 多跳问题：依赖agenticrag
